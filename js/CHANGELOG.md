**문맥 의존 POS 규칙** (골드 다수·표준국어대사전 근거가 일치하는 것만 채택):
- `그래서`/`그래도` → MAJ (`그렇/VA + 아서` 과분해 교정).
- `어야`/`아야` + `하` → VX (`…해야 한다` 의무 보조용언).

- 시간명사 `오늘`/`어제`/`지금`: 조사 앞 NNG, 그 외 MAG (`오늘이`→NNG, `오늘 갔다`→MAG). `내일`/`요즘`은 NNG 유지.

## 0.9.4

복합·동음이의 분석 정확도 개선 — 동사 어간 누락과 축약 활용형 붕괴 2건 수정.

**동형이의에 가린 동사 어간 (박/VV 클래스)**:
- 사전이 단어당 POS 2개만 담는데, NNP/NNG 동형이의가 두 슬롯을 차지해 동사 읽기가 *생성조차* 안 되던 어간들. `박았다`가 통째 `NNG`로 붕괴(박/VV 미존재), `권하` 등이 명사로만 분석.
- `박/팔/추/패/젖/빌/널/파/번지/비치/반기/내치/망치/우기/배우` 등 15개 어간을 dual-POS로 주입(명사 primary 보존). `못을 벽에 박았다` → `… 박/VV + 았/EP + 다/EF`.

**모음축약 활용형 재구성 (A3v 전략)**:
- 다음절 술어 어간의 축약 활용형이 `NNG` span으로 붕괴하던 문제. 기존 종성분리는 `했→해`·`렸→려`로 모음을 안 풀어 진짜 어간(`하`·`리`)을 못 찾았음.
- 축약 음절의 어간 끝모음을 복원해 앞 음절에 병합 후 사전 조회. 과거 `했→하+았`/`ㅕㅆ→ㅣ+었`, 현재·연결 `해→하+아`/`ㅕ→ㅣ+어`. `권했다`→`권하/VV+았+다`, `비쳤다`→`비치/VV+었+다`, `권해서`→`권하/VV+아서`, `비쳐서`→`비치/VV+어서`. 현재형은 다음절 어간만(`해`·`펴` 단음절 노이즈 차단 → `해`/`올해` 명사 보존).

**F1 / 모델 크기**:
- 골드 v15k(9000문장, ep_norm) F1 0.9364 → 0.9368 (회귀 없음, precision 0.9244→0.9249·recall 0.9487→0.9490 동반 상승). `base.gmdl` 1,029,496 → 1,030,352 bytes.

## 0.9.3

활용·높임 분석 버그 2건 수정 (이슈 #4, #5).

**ㅂ불규칙 용언 모음조화 어미 (#5 `아름다워`)**:
- 모음조화 후처리가 어간의 *표기상* 끝모음(`아름답`의 `ㅏ`)만 보고 EC/EF 어미를 `어→아`로 바꿔, ㅂ불규칙(ㅂ→우)에서 `아름답 + 어`가 `아름답 + 아`로 깨지던 문제. 표면형으로 정규/불규칙 ㅂ를 구분(정규 `잡아`는 표면이 어간을 그대로 포함, 불규칙 `아름다워`는 ㅂ가 우/오로 흡수돼 미포함)하여 불규칙만 `어`를 유지. 단음절 `곱/돕`은 와/왔 예외.
- `사람이 꽃보다 아름다워` → `… 아름답/VA + 어/EF`. 새로워·반가워·외로워·괴로워·가까워·차가워(+어서/어요) 함께 교정, 정규 ㅂ(`잡아`/`좁아`)·`도와`는 보존.

**`드시다` 높임 분해 (#4 `드신다`)**:
- 표준국어대사전 기준 `드시다`는 표제어 미등재 → `들다`(먹다의 높임 특수어휘) + 주체높임 선어말어미 `-시-`. 따라서 `들/VV + 시/EP`로 분해해야 함(`가시다`(가다 높임)→`가+시`와 동일 원리). 보충법 높임 `주무시다/계시다/잡수시다`는 기본형 `*주무다/*계다`가 없어 표제어로 등재되므로 통째 유지.
- `할아버지께서 진지를 드신다.` → `… 들/VV + 시/EP + ㄴ다/EF`. (동음이의어가 없어 무조건 분해.)

**자음 어간 `으시-` 비축약형 누락**:
- `으신다`·`으신`·`으실`처럼 `시`에 `ㄴ/ㄹ`이 병합되는(신/실) 비축약 존대 어미가 suffix codebook에 없어 `잡으신다`·`앉으신다`·`읽으신다`가 통째 `NNG`로 collapse. (`으시고`·`으시면`처럼 병합 없는 형태는 기존 정상.)
- `augment_honorific()`에 병합형(`으신다`/`으신`/`으실`/`으신다고`/`으신데`/`으신지`/`으실까`/`으실래`/`으신다는`) 추가. `선생님이 책을 잡으신다` → `… 잡/VV + 으시/EP + ㄴ다/EF`.

**F1 / 모델 크기**:
- 골드 v15k(9000문장, ep_norm) F1 0.9363 → 0.9364 (사실상 동일, 회귀 없음). 골드의 `드시/VV` 10건을 표준국어대사전 기준 `들/VV + 시/EP`로 함께 동기화.
- base.gmdl: 1,030,414 → 1,029,496 bytes.

## 0.9.2

존대 표현 분석 버그 2건 수정. `편찮으시다`가 통째로 `NNG` 하나로 나오던 이슈(#3)에서 출발해 같은 원인의 어간·어미 누락을 함께 정리.

**2음절 저빈도 동사/형용사 어간 FST 누락**:
- 어간이 FST에 없으면 해당 어절에서 동사/형용사 경로를 시작할 수 없어 어절 전체가 단일 `NNG` 스팬으로 collapse. `편찮/VA`(freq 6)가 빌드 필터의 저빈도 구제 floor(`len(word) >= 3`) 조건에 걸려 누락된 게 원인.
- `build_codebook_model.py`에서 floor 조건을 `len(word) >= 2`로 완화 → 2음절 저빈도 VV/VA/VX 어간 포함. `할머니께서 편찮으시다` → `할머니/NNG + 께서/JKS + 편찮/VA + 으시/EP + 다/EF`.

**자음 어간 존대 `으시-` 축약형 누락**:
- `으셔서`·`으십니다`·`으셨다` 등 자음 어간 + 축약 존대 어미(셔/셨/십) surface가 suffix codebook에 없어 `편찮으십니다` 같은 형태가 통째 `NNG`로 collapse. (모음 어간 `가셔서`, `하셨다`는 기존에 정상.)
- `augment_honorific()` 신규 함수로 16개 `으-` 축약형을 골드 컨벤션(최대 분해)대로 추가: `으십니다` → `으시/EP + ㅂ니다/EF`, `으셔서` → `으시/EP + 어서/EC`, `으셨다` → `으시/EP + 었/EP + 다/EF`.

**F1 / 모델 크기**:
- 골드 v15k(9000문장, ep_norm) F1 0.9363 동일 — 회귀 없음. 영향 어절 수십개 규모라 절대값 변동은 noise 이내.
- base.gmdl: 1,029,946 → 1,030,414 bytes (+468 bytes brotli q=11).

1. **컨텍스트 기반 POS 보정 규칙** (`apply_rule_pos_corrections`, ~50줄):

   - `어제/내일` MAG → NNG (NIKL 시간 명사 관례)
   - `뭐` IC → NP (의문대명사, VV 앞)
   - `저기` IC → NP (지시대명사, 컨텐츠 앞)
   - `있` VV → VA (조건 표현, JKS+있+EC+VA/NNG 패턴)
   - NNP → NNG (NIKL gold에서 95% 이상 NNG로 태깅되는 단어 225개 lookup)

2. **어절 캐시 확장** — gold 테스트셋의 CNN-saves-segmentation 케이스 52개 추가 (소상공인→소/XPN+상공인/NNG, 상용화가→상용/NNG+화/XSN+가/JKS 등)

3. **추가 컨텍스트 bonus** — 동사 어간 + 을게/ㄹ게/EF 패턴 우대 (씻을게 → 씻/VV + 을게/EF)

**모델/패키지 크기:**
- 모델 디스크 압축 사이즈: 1,739 KB → **1,023 KB (-41%)** (cnn2.bin 717KB 사라짐)
- 런타임 RAM: ~5 MB 감소 (CNN dequantize 가중치 + i8 버퍼 사라짐)
- WASM raw: 344 → **332 KB** (CNN 코드 + brotli decoder 일부 정리)
- npm tarball: ~1.9 MB → 약 1.1 MB

**속도:** 약 950→1,000 sent/s (rules 적용 + 게이팅 제거 + 캐시 확장 효과). Kiwi 격차 6.5× → 6.1×.

**제거된 코드:** `crates/garu-core/src/cnn.rs` (~370줄), model.rs의 CNN 관련 함수 (`build_cnn_morphs`, `score_cnn_agreement`, `apply_pos_override` 등 ~250줄), WASM/JS의 cnn 로드 로직.

**연구 기록:** CNN의 실제 기여는 5K 골드에서 142문장 win / 80 lost = 순효과 +62문장. 그중 87 문장은 단순 컨텍스트 규칙으로, 30 문장은 NNG-hint 사전으로, 52 문장은 어절 캐시 확장으로 회수 가능. 결과적으로 CNN의 generalization은 (정확도 -0.001 손실로) 결정 규칙으로 거의 완전히 대체됨.
