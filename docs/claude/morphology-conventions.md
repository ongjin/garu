# 형태소 분석 정답 기준

## 1. 정답 권위 순위 (골드/Kiwi를 맹신하지 말 것)

골드셋과 Kiwi는 하나의 **말뭉치 컨벤션**일 뿐 언어학적 정답이 아니다. GitHub 이슈 등 **전문가 제보가 골드/Kiwi와 충돌하면**, 골드를 ground truth로 가정하지 말고:

1. 인터넷에서 **표준국어대사전 / 국립국어원 온라인가나다**로 진짜 답을 확인한다 (WebFetch/WebSearch).
2. 표준대사전 기준이 골드와 다르면 **분석기 + 골드 둘 다** 그 기준으로 교정한다 (한쪽만 고치면 F1 불일치).

> 실제 사례: issue #4에서 골드/Kiwi는 `드시/VV`였지만 표준국어대사전 기준 `들+시`가 맞아 골드(10건)가 틀렸음이 드러남. 사용자 명시 지침: "골드셋과 Kiwi를 100% 신뢰하면 안 돼. 전문가 제보라 신뢰성이 높다."

## 2. 높임 선어말어미 -시- 분리 원칙

**기본형이 따로 존재하면 -시- 분리, 보충법(기본형 없음)이면 통째.** 판별 기준 = 표준국어대사전 표제어 등재 여부.

| 동사 | 표준대사전 표제어 | 분석 | 근거 |
|------|------------------|------|------|
| 드시다 | ✗ 미등재 | **들/VV + 시/EP** | 들다(먹다의 높임 특수어휘) 존재 |
| 가시다(가다 높임) | ✗ (등재된 가시다는 "사라지다") | **가/VV + 시/EP** | 가다 존재 |
| 주무시다 | ✓ 등재 | 주무시/VV (통째) | *주무다 없음 (보충법) |
| 계시다 | ✓ 등재 | 계시/VV (통째) | *계다 없음 |
| 잡수시다 | ✓ 등재 | 잡수시/VV (통째) | 어휘화 |

- **드시다 = 들/VV + 시/EP** 채택 (issue #4, commit `fe0b6e7`). 옛 골드 클렌징(kiwi 따라 드시/VV)을 뒤집고 골드 10건 교정. 구현: `fix_deusi_honorific`(드시는 동음이의어 없어 무조건 분해).
- **미해결**: `가신다`는 `가다`높임(가+시) vs `가시다`(맛/냄새가 사라지다, 표제어 등재)가 **동음이의** → 문맥 의존이라 자동분해 안 하고 `가시/VV` 유지. `맛이 가신다`는 맞지만 `선생님이 가신다`는 못 잡음.

## 3. ㅂ불규칙 모음조화

ㅂ불규칙은 ㅂ→우로 실현되어 어미가 **`-어`(워)**. 단음절 **곱/돕만 `-아`(와)** 예외 (한글맞춤법 제18항). 아름답→아름다워, 새롭→새로워, 곱→고와, 돕→도와.
- issue #5 (commit `0c32caf`): 모음조화 후처리가 어간 표기상 끝모음(ㅏ/ㅗ)을 보고 `어→아`로 깨뜨리던 것을, **표면형으로 정규/불규칙 구분**해 가드 (`잡아`⊃`잡` 정규 / `아름다워`⊅`아름답` 불규칙). 상세는 [analyzer-internals.md](analyzer-internals.md).

## 4. 자음어간 으시 (OOV)

자음어간 + `시`에 ㄴ/ㄹ이 병합되는 비축약형(`으신다`/`으신`/`으실`)은 접미사 코드북에 없으면 어절 통째 NNG로 붕괴. `build_codebook_model.py`의 `augment_honorific()`에 추가 (모델 리빌드 필요). 비병합형(으시고/으시면)은 기존 정상.

## 5. ep_norm (평가 정규화)

`eval_f1.py`의 ep_norm은 측정 아티팩트를 제거: jamo, **모음조화(아↔어)**, EP축약, 여불규칙, XR결합, 태그 정규화. 그래서 **모음조화 수정(#5)은 norm F1엔 안 잡힌다** (양쪽을 같게 봄). 실제 표면 영향은 `--no-norm`(raw)으로 확인. 헤드라인 F1은 norm=True 기준 93.6%.
